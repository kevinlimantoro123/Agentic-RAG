# Agentic Clinical RAG on InterSystems IRIS

An **agentic** Retrieval-Augmented Generation system for clinical documents. Upload a
clinical PDF, and the app extracts вҶ’ cleans вҶ’ chunks вҶ’ embeds it into an **InterSystems
IRIS** vector store (with an **HNSW** index), then lets you ask clinical questions. A
GPT-4o agent decides *which tools to call and when* вҖ” searching patient records and/or
authoritative medical guidelines вҖ” looping until it can answer with citations.

## Key features

- **Agentic loop** вҖ” GPT-4o uses OpenAI function calling (`tool_choice="auto"`) to choose
  tools dynamically and iterate (up to 10 steps), instead of a fixed retrieve-then-answer chain.
- **Two tools** the agent can call:
  - `retrieve_patient_records` вҖ” semantic vector search over clinical notes (HNSW).
  - `guideline_search` вҖ” authoritative web sources (NICE, FDA, CDC, ACE, NDF, HSA, NIH) via Tavily.
- **IRIS vector store** with a managed **HNSW** index and server-side embedding generation.
- **End-to-end PDF pipeline** вҖ” extraction (text/tables/images), cleaning, heading-aware
  chunking with patient/visit-date metadata, and bulk load.
- **Streamlit UI** with live agent step tracing and `[P#]` / `[G#]` citations.

## Tech stack

| Layer            | Technology                                                  |
|------------------|-------------------------------------------------------------|
| LLM / agent      | OpenAI GPT-4o (function calling)                            |
| Embeddings       | OpenAI `text-embedding-3-small` (1536-dim), generated by IRIS |

| PDF processing   | `unstructured`, `pdfplumber`, `pymupdf`, Tesseract OCR, Poppler |
| Chunking         | LangChain text splitters + `tiktoken`                       |
| UI               | Streamlit                                                   |

## Prerequisites

- **Python 3.10+** (3.12 recommended)
- **Docker Desktop** (to run IRIS)
- **Tesseract OCR** вҖ” required by the `hi_res` extraction strategy
- **Poppler** вҖ” required by `pdf2image` for PDFвҶ’image conversion during extraction
- **API keys**: OpenAI and Tavily

> **Why Tesseract + Poppler?** The extractor uses `unstructured`'s `hi_res` strategy, which
> rasterizes each PDF page (Poppler) and runs layout/OCR (Tesseract) to capture tables and
> scanned content. Without them, extraction of non-trivial PDFs fails.

## Setup

### 1. Python environment

```powershell
cd Agentic-RAG
python -m venv venv
.\venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

> **Import path tip:** the app imports the `rag_tool` and `pipeline` packages from the
> project root. If you hit `ModuleNotFoundError: No module named 'rag_tool'`, either run
> Streamlit from the project root, or add a `.pth` file pointing at the root into your
> venv's `site-packages`.

### 2. InterSystems IRIS (Docker)

Run IRIS Community Edition with a **named volume** so your data survives container restarts:

```powershell
docker run -d `
  --name iris-rag `
  -p 1972:1972 `
  -p 52773:52773 `
  -v iris-rag-data:/usr/irissys/mgr `
  intersystemsdc/iris-community:2026.2
```

Wait until `docker ps` shows the container as `(healthy)`. The Management Portal is at
<http://localhost:52773/csp/sys/UtilHome.csp> (default login `SuperUser` / `SYS`).

> **Persistence:** everything IRIS-side (namespace, configs, loaded data) lives under
> `/usr/irissys/mgr`, which is in the `iris-rag-data` volume. `docker stop`/`start` and even
> `docker rm` + recreate (with the same `-v`) keep your data. Only `docker volume rm
> iris-rag-data` or `docker compose down -v` wipes it.

### 3. Create the RAG namespace

In the Management Portal: **System Administration вҶ’ Configuration вҶ’ System Configuration вҶ’
Namespaces вҶ’ New Namespace**. Name it `RAG`, accept the default database paths, and save.

### 4. Create the SSL config (for OpenAI calls)

IRIS calls the OpenAI embedding API over HTTPS, which needs a TLS configuration named
`llm_ssl`:

```powershell
docker exec -i iris-rag iris session iris -U "%SYS"
```
```objectscript
do ##class(Security.SSLConfigs).Create("llm_ssl")
halt
```

### 5. Configure environment variables

Create a `.env` file in the project root:

```env
OPENAI_API_KEY=sk-proj-your-key
TAVILY_API_KEY=tvly-your-key
IRIS_CONN_STR=iris://superuser:sys@localhost:1972/RAG
```

### 6. Register the embedding config

This registers the embedding model IRIS uses for the `EMBEDDING(...)` columns. It is named
`bge-base-config` (kept for code compatibility) but points at OpenAI:

```powershell
.\venv\Scripts\python.exe setup_embedding_config.py
```

Expected output:
```
Registered embedding configs:
   bge-base-config | %Embedding.OpenAI | dim 1536
```

> The UI's **Reset Database** button clears the tables directly via `DELETE` (see
> `reset()` in `rag_tool/test_rag.py`) вҖ” no stored procedure setup is required.

### 7. Tesseract OCR path

`pipeline/extractor.py` locates the Tesseract binary automatically: it checks the
`TESSERACT_CMD` env var first, then your system `PATH`, then common install locations. In
most cases no action is needed. If auto-detection fails, set the env var, e.g.:

```env
TESSERACT_CMD=C:\Users\<you>\AppData\Local\Programs\Tesseract-OCR\tesseract.exe
```

(Install via `winget install UB-Mannheim.TesseractOCR`.)

## Running the app

```powershell
.\venv\Scripts\Activate.ps1
streamlit run frontend/app.py
```

Open <http://localhost:8501>.

1. **Upload a clinical PDF.** The pipeline runs automatically (extract вҶ’ clean вҶ’ chunk вҶ’
   load). On first load IRIS creates `Embedding.Clinical`, computes embeddings via OpenAI,
   and builds the HNSW index. Duplicate uploads are detected by SHA-256 hash.
2. **Set query filters** in the sidebar (PDF, optional patient, optional visit date,
   preferred guideline source, top-k).
3. **Ask a question** and click **Run Agent.** Expand **рҹ”§ Agent Steps** to watch which
   tools the agent chose and what each returned.

**Sample question that exercises both tools:**
> "Is this patient's current metformin dose within the recommended range, and are there any
> contraindications given their kidney function?"

## How it works

### The agentic loop (`rag_tool/agent.py`)

A single `SYSTEM_PROMPT` defines the agent. Each turn, GPT-4o is given both tools with
`tool_choice="auto"`. It may call `retrieve_patient_records`, `guideline_search`, both, or
neither вҖ” then sees the results and decides whether to refine, call again, or answer. The
loop runs up to `max_iterations` (default 10). `run_agent_streaming` yields events
(`tool_start`, `tool_result`, `answer`) for the live UI trace.

> A purely factual question (e.g. "what meds is the patient on?") will correctly use only
> **one** tool вҖ” that's the agent avoiding unnecessary work, not a bug. Questions that need
> external evidence trigger `guideline_search` as well.

### HNSW vector search (`pipeline/loader.py`, `rag_tool/test_rag.py`)

The loader creates:

```sql
CREATE INDEX HNSWIndex ON TABLE Embedding.Clinical (DescriptionEmbedding)
AS HNSW(M=32, efConstruction=100, Distance='DotProduct')
```

Retrieval issues a plain `TOP-k ... ORDER BY VECTOR_DOT_PRODUCT(...) DESC` query (no
`DISTINCT`) so the IRIS optimizer is *eligible* to use the HNSW index; deduplication happens
in Python afterward.

> **HNSW at small scale:** IRIS's cost-based optimizer only chooses the HNSW (approximate)
> index when the table is large enough that approximate search beats an exact scan. With a
> few dozen rows it will correctly do an exact linear scan. Confirm the chosen plan with
> `EXPLAIN <query>`.

## Troubleshooting

| Symptom | Cause / fix |
|---|---|
| `ModuleNotFoundError: No module named 'rag_tool'` | Run from the project root, or add a `.pth` with the root path to `venv/.../site-packages`. |
| `TesseractNotFoundError` | Set `TESSERACT_CMD` in `.env` (or install Tesseract on PATH). |
| `pdf2image` / Poppler errors | Install Poppler and ensure its `bin` is on PATH. |
| `numpy` fails to build during `pip install` | Use a Python version with prebuilt wheels, or `pip install numpy --only-binary :all:`. |
| `Embedding configuration bge-base-config ... is not defined` | Run `setup_embedding_config.py` against the `RAG` namespace. |
| `<COMMUNICATION LINK ERROR>` on insert/embedding | The embedding call failed server-side вҖ” verify the `llm_ssl` SSL config exists and `OPENAI_API_KEY` is valid. |
| `Unable to allocate a license` (connection refused) | IRIS Community caps concurrent connections. Close extra clients or `docker restart iris-rag` to clear stale slots. |
| IRIS won't start after recreate / CPF parse error | Stale volume from an older image. Remove the volume and recreate (you'll need to redo namespace + config setup). |

## Notes & caveats

- **`bge-base-config` is a label, not the model.** The project originally used a local BGE
  SentenceTransformers model, but loading torch inside the IRIS process caused segfaults, so
  embeddings were switched to OpenAI `text-embedding-3-small`. The config name was kept so no
  application code needed to change. Rename it across `loader.py`, `test_rag.py`, and
  `setup_embedding_config.py` if you want the name to read truthfully.
- **Clinical text is sent to OpenAI** for both embedding and answering. Use only with data
  you're authorized to send to a third-party API.
- **Recovery after losing the volume:** recreate the `llm_ssl` SSL config (step 4), re-run
  `setup_embedding_config.py` (step 6), then re-upload your PDFs. Host-side files (`.env`,
  code, `venv`) are unaffected.
